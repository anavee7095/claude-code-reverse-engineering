Core Flow
Tool Execution Pipeline

Every user query flows through this pipeline — from input to tool execution to streaming response.

1
User Input → QueryEngine
System prompt assembly, message normalization, beta headers, tool schema build with defer_loading.
2
LLM API Streaming
4 backends (Anthropic / Bedrock / Azure / Vertex). Retry with backoff. 90s idle timeout. Token counting.
3
Permission Check
4-step pipeline + auto-mode classifier. 7 modes. deny always beats allow. 23 bash security validators.
4
Tool Execution
StreamingToolExecutor: concurrent-safe tools run in parallel, unsafe ones exclusive. Bash error cascades siblings.
5
Result → Next Turn
ToolResult{data, newMessages?, contextModifier?, mcpMeta?} feeds back to LLM. 8-stage recovery on failure.

Implementation Warnings
Critical Caveats

19 WILL_CAUSE_BUG-level issues documented. These are the most dangerous ones.

WILL_CAUSE_BUG
401 Token Refresh Loop
No max refresh count in withRetry(). Invalid tokens retry up to MAX_RETRIES=10 with same broken token.
WILL_CAUSE_BUG
Settings Array Merge
deny + allow arrays concat (not override). Policy deny and user allow both exist in merged result.
WILL_CAUSE_BUG
TOCTOU Path Validation
Symlink can be swapped between validation and file access. Shared /tmp is exploitable.
WILL_CAUSE_BUG
Hook Input Not Chained
PreToolUse hooks run in parallel via Promise.all(). Each sees original input, not previous hook's output.
WILL_CAUSE_BUG
No Ink Error Boundary
Custom reconciler has no error boundary. Component render throw crashes entire TUI.
CONFUSION
React 19 Concurrency Off
maySuspendCommit() always returns false. Suspense and useTransition() are silently disabled.
